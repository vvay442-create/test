# AGENTS.md

Правила проекта для AI-агентов (Google Jules, SourceCraft CLI, Claude Code и др.).
Соблюдай их в каждой задаче.

## Стек

- Чистые **HTML + CSS + JS**, без фреймворков и без шага сборки.
- Сайт открывается двойным кликом по `index.html` и работает на GitHub Pages.
- Все пути — **относительные** (`styles.css`, `data.js`, а не `/styles.css`): Pages
  раздаёт сайт из подкаталога `/<имя-репо>/`.
- Никаких внешних зависимостей по сети: шрифты системные, ассеты — локальные.
- `data.js` и `app.js` подключаются как **классические** `<script>` и используют
  общие глобальные переменные. Не переводи их на ES-модули (`import`/`export`) —
  иначе сломается открытие двойным кликом по `file://`.

## Данные

- Вехи хронологии живут только в `data.js` (массив `MILESTONES`).
- Добавляешь или меняешь веху — правишь `data.js`, а не разметку.
- Каждая веха обязана иметь поля: `year` (число), `title`, `description`, `tag`.
- `tag` должен входить в список `PARADIGMS` из `data.js`. Под новую парадигму
  сначала добавь тег в `PARADIGMS`.

## Изменения и тесты

- Любое изменение поведения сопровождается тестом в `tests/`.
- Тесты осмысленные, быстрые и понятные по именам — они видны в логах агента как
  петля верификации.
- Перед коммитом обязательно прогоняй:

  ```bash
  npm test
  npm run lint
  ```

- Коммить только при зелёных тестах и линте.

## Чего не делать

- Не трогай файлы в `.github/workflows/` — CI и деплой уже настроены.
- Не добавляй фреймворки, бандлеры и сетевые CDN-зависимости.
- Не переноси данные из `data.js` в разметку.
